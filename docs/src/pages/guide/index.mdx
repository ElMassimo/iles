## New îles Project 💿

Run the following command to create a new îles project using the [starter] template.

```bash
npm create iles@latest # or pnpm, yarn, bun
```

## Add îles to Existing Project

To add îles to your existing Vite-powered Vue project, refer to the [manual installation](#manual-installation) section. 

New to Vue? Scaffold a new Vue project using <kbd>npm create vue@latest</kbd> and add îles to it. 

## Usage 🚀

Open your project folder in Visual Studio Code and using the integrated terminal,

Install your dependencies:

```bash
npm install # or pnpm, yarn, bun
```
<Tip>
Learn about the [project structure] to simplify and optimize your project.
</Tip>

Start your development server:

```bash
npm run dev # or pnpm, yarn, bun
```

Build for production:
<Tip>
Remember to provide `client:` [strategy] for interactive components that requires client-side hydration before building for production.
</Tip>

```bash
npm run build # or pnpm, yarn, bun
```

Preview your site locally after building:

```bash
npm run preview # or pnpm, yarn, bun
```

## Manual Installation

In your Vite-powered Vue project, install `iles` using:

```bash
npm add -D iles # or pnpm, yarn, bun
```

### 1. Remove `index.html`

îles automatically loads a shell page with it's own entry shell app (for an enhanced developer experience), so ensure you remove/archive the `index.html` file from your project root. 

For eg, to archive, rename `index.html` to `!index.html`.

### 2. Add `iles.config.ts`

Create an `iles.config.ts` file (mandatory) at your project root like this. 

```ts
import { defineConfig } from 'iles'

export default defineConfig({
  siteUrl: 'https://myawesomeidea.com', // Your site URL
  modules: [
    // Add iles modules here, refer to Plugins page
  ],
  vite: {
    plugins: [
      // Add vite plugins here, for eg, vite-plugin-vue-devtools
    ]
  }
})
```

### 3. Migrate `vite.config.ts`

If you have an existing Vite configuration file, such as `vite.config.ts`, migrate your Vite configuration into the `vite` key of your `iles.config.ts` file as shown below. 

<b>Important</b>: Skip migrating `@vitejs/plugin-vue` (Vite plugin for Vue) as îles already ships it. Ensure it's not added as a Vite plugin in the `iles.config.ts` file.

Once you have migrated your Vite configuration, ensure to remove/archive `vite.config.ts` file so that it doesn't interfere with îles development server workflow. 

For eg, to archive, rename `vite.config.ts` to `!vite.config.ts`.

```ts
import { defineConfig } from 'iles'

export default defineConfig({
  siteUrl: 'https://myawesomeidea.com', // Your site URL
  modules: [
    // Add iles modules here, refer to Plugins page
  ],
  vite: {
    // Vite configuration goes here
    plugins: [
      // Add vite plugins here, for eg, vite-plugin-vue-devtools
    ]
  }
})
```

### 4. Add `app.ts`

Create an `src/app.ts` (mandatory) file like this.

```ts
import { defineApp } from 'iles'

export default defineApp({
  enhanceApp ({ app, head, router }) {
    // Configure the shell app to customise with development/build logic
  },
})
```

### 5. Add `site.ts`

Create an `src/site.ts` (mandatory) file like this:

```ts
export default {
  title: 'Cafe Tee Kaapi',
  description: 'Sip, Savor, and Spark Ideas!',
}
```

### 6. Add `layouts`

Create your default layout (or convert your existing one) as src/layouts/default.vue like this:

```vue
<script setup lang="ts">
// const { frontmatter, meta, site, route } = usePage()
// site-wide logic
</script>

<template>
  <div>
    <header>
      <!-- NavBar vue island with client hydration strategy -->
      <!-- SearchBar solid island with client hydration strategy -->
    </header>
    <main>
      <!-- Pages are typically static with few dynamic islands -->
      <!-- Pages using this default layout will load in this slot -->
      <slot />
    </main>
    <footer>
      <!-- TheFooter static vue component with no interactions -->
    </footer>
  </div>
</template>

<style>
body {
  margin: 0;
  height: 100vh;
}
</style>
```

### 7. Add `pages`

- If you don't have any pages yet, create your home page as `src/pages/index.vue` as shown below:

<Tip>
îles uses the `pages` and `layouts` folder conventions with filenames in kebab-case.
- Rename the folder that contains all your pages (for eg, `src/views`) to `src/pages`.
- Rename the `.vue` file of your home page (for eg, `HomeView.vue`) to `index.vue`.
- Rename the `.vue` files of your other pages to follow kebab-case. For eg, `AboutView.vue` could be renamed to `about-us.vue`
- Create your default layout (or convert your existing one) as `src/layouts/default.vue` like this: 
</Tip>

```vue
<page>
  title: Welcome to Cafe Tee Kaapi
  description: Crafting Memories, Brewing History!
</page>

<script setup lang="ts">
const { frontmatter } = usePage()
const { title, description } = frontmatter
</script>

<template>
  <section class="hero">
    <div class="card">
      <h1>{{ title }}</h1>
      <p>{{ description }}</p>
      <small>- Built with îles</small>
    </div>
  </section>
</template>

<style scoped>
.hero {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh;
  background: url('https://images.unsplash.com/photo-1509042239860-f550ce710b93?crop=entropy&cs=tinysrgb&fit=max&fm=jpg&ixid=M3wyMDkyMnwwfDF8c2VhcmNofDF8fGNhZmV8ZW58MHx8fHwxNjg2MTkzMDk2fDA&ixlib=rb-4.0.3&q=80&w=2000%20768w')
    no-repeat center/cover;

```

### 7. Add îles executables

Add the `iles` executables in your `scripts` section of your `package.json`.

```json
  "scripts": {
    "dev": "iles dev",
    "build": "iles build",
    "preview": "iles preview --open"
  },
```

### 8. Other Updates: 

Archive your root component (for eg, `App.vue`), entry file (for eg, `main.ts`) and your `router` folder.
  - îles supports file-based routing, so your `router` folder is not required. Any custom logic might have to be migrated to `app.ts`.
  - Contents of your root component (for eg, `App.vue`) and your entry file (for eg, `main.ts`) might have to be migrated to either your default layout, one of your Islands, or `src/app.ts`.

If you migrated from a previous application, test your project to ensure it functions correctly. After a successful migration, you can safely remove your previous files such as your previous shell page and Vite configuration file (for eg, `!index.html` and `!vite.config.ts` from above steps).

Well Done! Now, follow to the [Usage](#usage) section to start your development server.
