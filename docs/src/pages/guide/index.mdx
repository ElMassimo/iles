## New îles Project 💿

Run the following command to create a new îles project using the [starter] template.

```bash
npm create iles@latest # or pnpm, yarn, bun
```

## Add îles to an Existing Project

To add îles to your existing Vite-powered Vue project, refer to the [manual installation](#manual-installation) section. 

New to Vue? 💚 Scaffold a new Vue project using the following command. Then, add îles to it.

```bash
npm create vue@latest # or pnpm, yarn, bun
```

## Usage 🚀

Open your project folder in Visual Studio Code and, using the integrated terminal,

### Install your dependencies:

```bash
npm install # or pnpm, yarn, bun
```

### Start your development server:

```bash
npm run dev # or pnpm, yarn, bun
```

### Build for production:

```bash
npm run build # or pnpm, yarn, bun
```

### Preview your site locally after building:

```bash
npm run preview # or pnpm, yarn, bun
```

### Alternatively, Build and Preview with a single command:

```bash
npm run now # or pnpm, yarn, bun
```

## Manual Installation

In your Vite-powered Vue project, install `iles` using:

```bash
npm add -D iles # or pnpm, yarn, bun
```

### 1. Remove `index.html`

îles automatically loads a shell page with its own entry shell app to enhance the developer experience.

Ensure you remove the `index.html` file from your project root to avoid conflicts with îles entry shell app.

### 2. Add `iles.config.ts`

Create an `iles.config.ts` file (mandatory) at your project root as shown below. 

```ts
import { defineConfig } from 'iles'

export default defineConfig({
  siteUrl: 'https://myawesomeidea.com', // Your site URL
  modules: [
    // Add iles modules here, refer to Plugins page
  ],
})
```

### 3. Existing `vite.config.ts`

If you have an existing `vite.config.ts` file, you can either keep it or migrate its configuration into the `vite` key of `iles.config.ts` as shown below.


<b>Important</b> Remove `@vitejs/plugin-vue` (Vite plugin for Vue)

- <b>Keeping</b> `vite.config.ts`: Remove `@vitejs/plugin-vue` from its plugins array to avoid conflicts, as îles already includes it.

- <b>Migrating</b> to `iles.config.ts`: Do not include `@vitejs/plugin-vue` in the migration. After migrating, remove `vite.config.ts` file from your project root.

```ts
import { defineConfig } from 'iles'

export default defineConfig({
  siteUrl: 'https://myawesomeidea.com', // Your site URL
  modules: [
    // Add iles modules here, refer to Plugins page
  ],
  vite: {
    // Vite configuration goes here
    plugins: [
      // Add vite plugins here, for example, vite-plugin-vue-devtools
    ]
  }
})
```

### 4. Add `app.ts`

Create an `src/app.ts` (mandatory) file as shown below.

```ts
import { defineApp } from 'iles'

export default defineApp({
  enhanceApp ({ app, head, router }) {
    // Configure the shell app with custom development/build logic
  },
})
```

### 5. Add `site.ts`

Create an `src/site.ts` (mandatory) file as shown below:

```ts
export default {
  title: 'Cafe Tee Kaapi',
  description: 'Sip, Savor, and Spark Ideas!',
}
```

### 6. Add `layouts`

Create your default layout (or convert your existing one) as `src/layouts/default.vue` (mandatory) as shown below:

```vue
<script setup lang="ts">
import { computed } from 'vue'
const { route } = usePage()

const navs = [
  { title: 'Home', to: '/' },
  { title: 'About Us', to: '/about-us' },
]
const currentPath = computed(() => {
  return route.path
})
// Other site-wide logic
</script>

<template>
  <div>
    <header>
      <nav>
        <!-- NavBar vue island with client hydration strategy -->
        <a
          v-for="({ title, to }, index) in navs"
          :key="index"
          :href="to"
          :class="{ active: currentPath === to }"
        >
          {{ title }}
        </a>
      </nav>
      <!-- SearchBar solid island with client hydration strategy -->
    </header>
    <main>
      <!-- Pages are typically static with few dynamic islands -->
      <!-- Pages using this default layout will load in this slot -->
      <slot />
    </main>
    <footer>
      <!-- TheFooter static vue component with no interactions -->
    </footer>
  </div>
</template>

<style>
body {
  font-family: Arial, sans-serif;
  margin: 0;
  height: 200vh;
}
nav {
  display: flex;
  justify-content: center;
  gap: 1rem;
  padding: 1rem;
}
a {
  text-decoration: none;
  font-weight: bold;
}
a.active {
  text-decoration: underline;
}
</style>
```

### 7. Add `pages`

If you don't have any pages yet, create your home page as `src/pages/index.vue` as shown below:

<Tip>
îles uses the `pages` and `layouts` folder conventions with filenames in kebab-case.
- Rename the folder that contains all your pages (for example, `src/views`) to `src/pages`.
- Rename the `.vue` file of your home page (for example, `HomeView.vue`) to `index.vue`.
- Rename the `.vue` files of your other pages to follow kebab-case. For example, `AboutView.vue` could be renamed to `about-us.vue`
</Tip>

```vue
<page>
  title: Home
  description: First Coffee, Then Conquer!
</page>

<script setup lang="ts">
const { site, frontmatter, meta } = usePage()

const { title: siteTitle } = site
const { description } = frontmatter
const { lastUpdated } = meta
</script>

<template>
  <section class="hero">
    <div class="card">
      <h1>Welcome to {{ siteTitle }}</h1>
      <p>{{ description }}</p>
      <p>Site last updated: {{ lastUpdated.toDateString() }}</p>
      <small>- Built with îles 💚</small>
    </div>
  </section>
</template>

<style scoped>
.hero {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 60vh;
}

.card {
  text-align: center;
  padding: 2rem;
  background: #fff;
  border-radius: 8px;
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
  border: 1px solid #ddd;
}

.card h1 {
  font-size: 2rem;
  margin-bottom: 0.5rem;
}

.card p {
  font-size: 1rem;
  color: #555;
}
</style>
```

îles provides excellent support for authoring your pages with markdown (`.md`, `.mdx`). Create a second page `src/pages/about-us.md` as shown below.

```md
---
title: About Us
description: Crafting Memories, Brewing History!
---

# This an about page

<style scoped>
main {
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  height: 60vh;
}
</style>
```

### 8. Add îles executables

Add the `iles` executables in your `scripts` section of your `package.json`.

```json
  "scripts": {
    "dev": "iles dev",
    "build": "iles build",
    "preview": "iles preview --open",
    "now": "npm run build && npm run preview"
  },
```

### 9. Root Component, Entry File & Router 

Remove or archive your root component and entry file (for example, `App.vue`, `main.ts`) as well as the `router` folder since îles uses file-based routing.

Any custom logic might need to be migrated to `app.ts`, your default layout, or one of your interactive Islands.


### 10. Remove Obsolete Files

If you migrated from a previous application, thoroughly test your project to ensure it functions correctly. After a successful migration, remove your obsolete files such as the shell page and Vite configuration file (`index.html` and `vite.config.ts` from above steps).
